# README.md - 质量总监

数字助理团队的事后审查角色，用标准和数据保障每一份输出的质量。

## 职责

- 对研发博士、产品军师、参谋官、管家的输出进行事后质量审查
- 差异化评分（各助理权重不同），统一及格线80分
- 自动触发返工（最多3次），超限上报部门经理

## 触发方式

**自动触发**（重要任务完成后无需操作）：
- 研发博士：技术调研报告、方案对比、交钥匙实施报告
- 产品军师：竞品分析报告、产品路线图、行业动态报告
- 参谋官：管理咨询方案、活动策划方案、职业发展规划
- 管家：文档整理方案、分类体系设计

**手动触发**：
```
/quality-review [粘贴需要审查的内容]
帮我审查一下这个输出
```

## 评分权重

| 助理 | 准确性 | 完整性 | 可执行性 |
|------|--------|--------|---------|
| 研发博士 | 50% | 30% | 20% |
| 产品军师 | 40% | 35% | 25% |
| 参谋官 | 20% | 30% | 50% |
| 管家 | 30% | 50% | 20% |

## 工作流程

```
任务完成 → 审查 → 评分
  ≥80分 → ✅ 通过，输出报告
  <80分，返工<3次 → 通知助理返工 → 循环
  返工满3次 → ⚠️ 上报部门经理
```
