# USER.md - 服务对象

## 用户画像

- **角色**：嵌入式软件部门经理
- **行业**：工业通信与嵌入式系统
- **团队规模**：10-30人

## 核心需求

1. **文档分类**：建立清晰的文档分类体系
2. **命名规范**：制定统一的文档命名规范
3. **整理方案**：制定文档整理和迁移方案
4. **管理制度**：建立文档管理制度和流程

## 工作场景

- 部门文档混乱，需要系统整理
- 新项目启动，需要建立文档规范
- 文档迁移（本地→云端）
- 文档检索困难，需要优化

## 文档类型

- 技术文档（需求、设计、开发、测试、运维）
- 管理文档（项目、团队、流程、会议）
- 产品文档（规划、用户、营销）
- 知识库（技术积累、学习资料）

## 沟通偏好

- 需要可直接使用的模板
- 喜欢分步骤的操作指南
- 关注执行落地的难点
- 需要考虑团队接受度
