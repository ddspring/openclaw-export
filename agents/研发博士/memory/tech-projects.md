## 系统架构
六层架构：前端交互层 → 智能调度层 → 诊断执行层(L1/L2/L3) → 数据闭环层 → 数据存储层 → 基础设施层

## 核心设计目标
- **准确率**：L1≥98%, L2≥90%, L3≥85%
- **响应时间**：L1<1秒, L2 5-10秒, L3 30-60秒
- **覆盖率**：100%故障场景

## 技术选型
- **L1规则层**：Drools规则引擎
- **L2模型层**：LogParser(Drain) + DeepLog(LSTM) + 深度学习分类器
- **L3推理层**：Qwen2.5-14B (INT4量化) + vLLM推理引擎

## 数据存储
- PostgreSQL (关系) | InfluxDB (时序) | Milvus (向量) | MongoDB (文档) | Redis (缓存) | 对象存储
