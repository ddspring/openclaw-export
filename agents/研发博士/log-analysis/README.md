# 日志智能分析

> ETBN故障诊断系统的日志分析模块

基于AI的日志智能分析系统，使用miniMax LLM进行日志分析和异常检测。

## 关联项目

本项目是 **ETBN混合智能故障诊断系统** 的一部分，负责日志读取与分析。

### 系统架构

```
ETBN故障诊断系统
├── L1: 规则层 - 处理简单故障
├── L2: 模型层 - 机器学习异常检测
├── L3: 推理层 - LLM深度分析
└── 📁 log-analysis (本项目) - 日志读取与分析
```

## 功能特性

- 📄 支持文件格式日志分析
- 🔍 智能异常检测
- 📊 自动生成分析报告
- 🚀 批量日志处理

## 项目结构

```
log-analysis/
├── config/          # 配置文件
│   └── config.yaml
├── logs/            # 日志样本
├── src/             # 源代码
├── reports/         # 分析报告
└── README.md
```

## 配置说明

主要配置项在 `config/config.yaml` 中：

- **LLM**: miniMax-CN API配置
- **日志路径**: G:\日志智能分析
- **日志格式**: 文件格式

## 快速开始

1. 修改 `config/config.yaml` 中的配置
2. 运行分析程序
3. 查看 `reports/` 目录下的分析报告
